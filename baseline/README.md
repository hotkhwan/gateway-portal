## Architecture Rules

### Route groups

| Group | Purpose | Auth required |
|---|---|---|
| `(app)` | Protected admin pages | ✅ JWT + X-Active-Org |
| `(base)` | BFF proxy + auth session endpoints | Server-only |
| `(public)` | Public pages | ❌ |

### Layer responsibilities

| Layer | Location | Responsibility |
|---|---|---|
| Page | `routes/(app)/*/+page.svelte` | UI only — call API lib, render state |
| Layout server | `routes/(app)/+layout.server.ts` | Auth guard, load session |
| API lib | `lib/api/*.ts` | All `fetch` calls to backend |
| Types | `lib/types/*.ts` | TypeScript interfaces |
| Stores | `lib/stores/*.ts` | Shared reactive state |
| BFF proxy | `routes/(base)/api/[...path]/+server.ts` | Transparent proxy + token refresh |

**Forbidden:**
- Direct `fetch` to backend from `+page.svelte` — use `lib/api/` ❌
- Business logic in `+page.svelte` — pages are UI only ❌
- Import server-only modules in client components (`lib/server/`) ❌
- Edit files under `src/lib/i18n/` or `src/paraglide/` — generated files ❌

## BFF Proxy Pattern

All API calls from the browser go through the SvelteKit server proxy at `/api/[...path]`.  
The proxy handles:
- Token injection from `session_token` cookie
- Proactive token refresh (when token expires within `AUTH_REFRESH_SKEW_SEC`)
- Transparent 401 → refresh → retry

```
Browser fetch("/api/v1/ingest/management")
  → SvelteKit BFF (+server.ts)
    → inject Authorization: Bearer <session_token>
    → forward to API_BASE + path
  → return response to browser
```

**Never** set `Authorization` header manually from the client — the BFF handles it.

```typescript
// ✅ correct — BFF adds auth automatically
const res = await fetch(`${BASE}/api/v1/ingest/management`)

// ❌ wrong — never set auth from client
const res = await fetch(url, { headers: { authorization: `Bearer ${token}` } })
```

## API Library Pattern

One file per domain in `src/lib/api/`.

```typescript
// src/lib/api/ingest.ts
import { PUBLIC_APP_BASE_PATH } from '$env/static/public'

const BASE = `${(PUBLIC_APP_BASE_PATH ?? '/aisom').replace(/\/$/, '')}/api`

// internal helper — not exported
async function apiFetch<T>(
  path: string,
  init?: RequestInit,
  orgId?: string
): Promise<ApiResponse<T>> {
  const res = await fetch(`${BASE}${path}`, {
    headers: {
      'content-type': 'application/json',
      ...(orgId ? { 'x-active-org': orgId } : {}),
      ...init?.headers
    },
    ...init
  })
  const json = await res.json()
  if (!res.ok) throw json
  return json as ApiResponse<T>
}

// exported domain functions
export async function listPendingEvents(orgId: string, params?: ListParams) { ... }
export async function approveEvent(orgId: string, eventId: string) { ... }
```

**Rules:**
- `apiFetch` is internal to each api file — not shared across files
- Every function that accesses org-scoped data passes `orgId` → `x-active-org`
- API functions throw `json` error object on non-ok response — pages catch and display
- No `console.log` in production api functions (debug only, remove before merge)

## Types Pattern

```typescript
// src/lib/types/ingest.ts

export interface PendingEvent {
  eventId: string
  orgId: string
  eventType: string
  rawBody: Record<string, unknown>
  fieldMappings: FieldMapping[]
  templateId?: string
  statusName: 'pending' | 'mapped' | 'approved' | 'rejected'
  sourceIp?: string
  createdAt: string   // RFC3339 UTC string (not Date — JSON)
  updatedAt: string
}

export interface FieldMapping {
  targetPath: string
  sourcePath: string
  transform?: string
  required: boolean
  confidence: number
  updatedAt: string
}

export interface MappingTemplate {
  templateId: string
  orgId: string
  name: string
  match: MatchRule
  mappings: FieldMapping[]
  createdAt: string
  updatedAt: string
}

export interface MatchRule {
  vendor?: string
  protocol?: string
  deviceType?: string
  subType?: string
  eventType?: string
}
```

**Rules:**
- Types mirror backend `ingestmod` models — keep in sync when backend changes
- Timestamps are `string` (RFC3339 from JSON) — convert to `Date` only in display layer
- Use `Record<string, unknown>` for `rawBody` — it's a freeform map
- Export a `transformBackendXxx()` function if backend shape differs from frontend type

## API Response Envelope

Backend always returns:

```typescript
// src/lib/types/index.ts
export interface ApiResponse<T> {
  code: string
  message: string
  status: boolean
  details: T
  pagination?: Pagination
}

export interface Pagination {
  page: number
  perPages: number
  totalRecords: number
  totalPages: number
  sortField: string
  sortOrder: 'asc' | 'desc'
}
```

Always use `r.details` (never `r.detail`) — backend standard uses `details`.

## Svelte 5 Runes — Rules

Use Svelte 5 runes syntax only. Do not use Svelte 4 reactive syntax.

```svelte
<script lang="ts">
  // ✅ Svelte 5 runes
  let events = $state<PendingEvent[]>([])
  let loading = $state(false)
  let error = $state<string | null>(null)

  // ✅ derived
  let pendingCount = $derived(events.filter(e => e.statusName === 'pending').length)

  // ✅ effect — use sparingly, avoid reactive loops
  $effect(() => {
    // runs when dependencies change
    loadEvents()
  })

  // ❌ Svelte 4 — do not use
  // let count = 0
  // $: doubled = count * 2
  // onMount(...)  ← use $effect instead
</script>
```

**`$effect` rules (critical — causes infinite loop if misused):**
- Never call a reactive state setter inside `$effect` that reads the same state
- Use `untrack()` for reads inside effects that should not be dependencies
- Prefer loading data in event handlers (`onclick`) over `$effect` when possible

## Auth Flow

```
/auth/session/start
  → set kc_state cookie
  → redirect to Keycloak authorize URL

Keycloak callback → /auth/callback
  → verify state
  → POST /api/auth/oauth (BFF)
    → backend exchanges code → access_token + refresh_token
  → set session_token + session_refresh cookies (httpOnly)
  → redirect to /dashboard

Protected page load
  → layout.server.ts reads session_token cookie
  → if missing → redirect to /auth/session/start

Logout
  → POST /auth/session/logout
    → call backend /auth/signout
    → delete session_token + session_refresh cookies
  → redirect to /
```

**Cookie names:**

| Cookie | Value | httpOnly |
|---|---|---|
| `session_token` | JWT access token | ✅ |
| `session_refresh` | Refresh token | ✅ |
| `kc_state` | OAuth CSRF state | ✅ |
| `return_to` | Post-login redirect path | ✅ |

## Active Org

Every org-scoped API call requires `X-Active-Org` header.  
Active org is stored in the `activeOrg` store.

```typescript
// src/lib/stores/activeOrg.ts
// read current org
import { activeOrg } from '$lib/stores/activeOrg'
const orgId = get(activeOrg)

// pass to api call
await listPendingEvents(orgId, params)
```

The BFF does NOT inject X-Active-Org automatically — the frontend must pass it per call.

## ENV Variables

| Var | Where | Purpose |
|---|---|---|
| `PUBLIC_APP_BASE_PATH` | client + server | Base path prefix (e.g. `/aisom`) |
| `PUBLIC_APP_BASE_PORT` | server | HTTP port |
| `API_BASE` | server only | Backend URL (e.g. `https://gateway.aisom.cloud/api/v1`) |
| `PUBLIC_KC_ISSUER_URI` | server | Keycloak issuer URL |
| `PUBLIC_KC_REALM` | server | Keycloak realm |
| `PUBLIC_KC_CLIENT_FE_ID` | server | Keycloak FE client ID |
| `AUTH_REFRESH_SKEW_SEC` | server | Token refresh lead time in seconds (default: 45) |
| `COOKIE_SECURE` | server | `true` in production (HTTPS) |
| `PUBLIC_APP_ORIGIN` | server | Override origin for redirect_uri (optional) |
| `AUTH_LOGOUT_PATH` | server | Backend logout path (default: `/auth/signout`) |

`PUBLIC_*` vars are safe to expose to the browser. Non-`PUBLIC_*` vars are server-only.

## Page Template

```svelte
<!-- src/routes/(app)/ingest/management/+page.svelte -->
<script lang="ts">
  import { m } from '$lib/i18n/messages'
  import { get } from 'svelte/store'
  import { activeOrg } from '$lib/stores/activeOrg'
  import { listPendingEvents } from '$lib/api/ingest'
  import type { PendingEvent } from '$lib/types/ingest'

  let events = $state<PendingEvent[]>([])
  let loading = $state(false)
  let error = $state<string | null>(null)
  let page = $state(1)
  let totalPages = $state(1)

  async function load() {
    loading = true
    error = null
    try {
      const orgId = get(activeOrg)
      const res = await listPendingEvents(orgId, { page })
      events = res.items
      totalPages = res.totalPages
    } catch (e: any) {
      error = e?.message ?? 'Failed to load'
    } finally {
      loading = false
    }
  }

  $effect(() => {
    load()
  })
</script>

<div class="container-fluid">
  <div class="d-flex justify-content-between align-items-center mb-4">
    <h1 class="h4 mb-0">{m.ingestManagementTitle()}</h1>
  </div>

  {#if loading}
    <div class="text-center py-5">
      <div class="spinner-border text-primary"></div>
    </div>
  {:else if error}
    <div class="alert alert-danger">{error}</div>
  {:else if events.length === 0}
    <div class="text-muted text-center py-5">{m.commonNoRecords()}</div>
  {:else}
    <!-- table -->
  {/if}
</div>
```

## Component Conventions

- Use Bootstrap classes — do not write custom CSS unless necessary
- Theme switching via `data-bs-theme="dark"` / `"light"` on `<html>` — use CSS variables, not hardcoded colors
- Leaflet maps: always use `showMap` flag + `requestAnimationFrame` for modal init (see `MapPicker.svelte` pattern)
- Icons: Bootstrap Icons (`bi bi-*`) or Iconify (`@iconify/svelte`)
